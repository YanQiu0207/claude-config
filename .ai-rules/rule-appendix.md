# 工程规则附录

本文件存放不适合长期内嵌在主规则中的模板和补充清单；主规则文件只保留决策、边界和强制动作。

## 1. 工程评审清单

- 关键路径日志是否完整，是否覆盖总耗时和关键步骤耗时。
- 报错日志是否足以定位问题，是否泄露敏感信息。
- 是否绕过防腐层直接耦合第三方 SDK 或模型。
- 是否把可调参数写死在代码里。

### 2.2 `stage-status.md`

用途：

- 记录每个阶段当前状态、评审结果、修复收口和放行结论

模板：

```md
# 阶段状态板

| 阶段 | 名称 | 状态 | 开发完成 | Review A | Review B | 已修完 | 阻塞原因 | 待清理 bug | 批准人 | 备注 |
|---|---|---|---|---|---|---|---|---|---|---|
| 0 | 示例阶段 | 开发 | 否 | 待开始 | 待开始 | 否 |  |  | 待填写 |  |

## 状态定义

- 未开始
- 开发
- 评审
- 修复
- 通过
- 阻塞
```

### 2.3 `agent-board.md`

用途：

- 仅在并行协作时使用，由主会话维护，记录各 agent 在做什么、当前状态、是否完成，以及它可写哪些文件。

模板：

```md
# Agent 分工板

| Task ID | Agent | 当前任务 | 写入范围 | 状态 | 是否完成 | 最近回传 |
|---|---|---|---|---|---|---|
| STAGE1-BE-001 | worker-a | 调整 `app/config.py` 配置加载 | `app/config.py` | 进行中 | 否 | 已开始修改，待回传 |

## 状态定义

- 待开始
- 进行中
- 阻塞
- 待集成
- 已完成
```
