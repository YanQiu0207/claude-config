当此 skill 生效时，回答第一行固定写：Using skill: english-learn

## 适用范围

- 输入是**一句**英文（也允许 2-3 句的小段，但范式提炼仍以核心一句为主）
- 用户希望从这句英文里抽出可复用的「句型范式」，能套到其它场景
- 默认行为：**先纠错 → 分析 → 自动写入** `E:/work/blog/英语/tips.md`，不再额外询问
- 不分析整段文章、不做发音/音标分析

## 输入

| 项 | 必填 | 说明 |
| --- | --- | --- |
| 英文句子 | 是 | 待分析对象，原样保留大小写与标点 |
| 潜台词/语境 | 否 | 帮助选准语气说明（如「想委婉指出对方文档不全面」） |

> 若输入句子有语法或用词错误，将**先在终端列出错误并给出修正后的句子**，然后基于修正后的句子继续后续流程。

## 处理流程

### 1. 语法 / 用法初检

在做任何范式提炼之前，对原句做一次快速检查：

- **硬错误**：语法错误、时态错误、主谓不一致、介词误用、拼写错误、可数/不可数错误
- **软问题**：用词生硬、中式英语、非自然搭配（不算错但母语者不会这么说）

判定结果分三种：

| 判定 | 处理 |
| --- | --- |
| 完全没问题 | 直接进入步骤 2，不输出纠错段落 |
| 仅有软问题 | 直接进入步骤 2，把更地道的说法留到步骤 4「美国人口语同义表达」中体现 |
| 有硬错误 | 先在终端输出「⚠ 语法/用法纠正」段落，然后用**修正后的句子**继续后续所有流程 |

「语法/用法纠正」段落格式：

```
⚠ 语法/用法纠正
原句：<original sentence>
问题：<逐条列出错误点>
修正：<corrected sentence>
说明：<一句话解释为何这样改>
```

后续步骤（拆解表、写入 tips.md 等）一律使用**修正后的句子**，原句仅出现在该纠正段落里，不写入文件。

### 2. 抽取核心范式

- 找出句子的骨架结构，写成 `[槽位 1] + [槽位 2] + ...` 的模板
- 槽位用方括号 + 简短中文标签，如 `[名词]`、`[核心信息]`、`[对象/范围]`
- 用反引号包整个模板：`` `That [名词] only [动词] the [核心信息] for [对象/范围].` ``
- 一句话点明范式的「言外之意」或「使用动机」（委婉指出 / 求步骤 / 主动声明范围 等）

### 3. 编写拆解表

固定 4 列：

| 槽位 | 作用 | 本句对应 |

把每个槽位填一行，本句对应词用反引号包裹。

### 4. 美国人口语同义表达

给出 **2-3 条**与（修正后的）原句**意思相同**、但更贴近**美国本土口语**的说法。

要求：

- 必须是美国母语者日常会脱口而出的表达，不是字典直译
- 每条注明**使用语境**与**与原句的语气差异**（更随意 / 更直接 / 更礼貌 / 更职场化 等）
- 至少覆盖 2 个不同语域（如：朋友闲聊 + 工作场合）

格式（落到 tips.md 的固定结构）：

```markdown
| 美国口语表达 | 语境 | 与原句的差异 |
| --- | --- | --- |
| `Native sentence A.` | 朋友间随便聊 | 更随意，省略主语 |
| `Native sentence B.` | 工作 / 邮件 | 更直接，少了委婉感 |
```

### 5. 语气梯度 / 近义对比（按需）

如果范式中有可替换的关键词（如 `only/just/merely`、`do/can/should`），给出梯度表：

| 词 | 语气 | 适用场景 |

如果没有此类替换点，本节可省略。

### 6. 套用例子

至少 2 个场景，最多 4 个，每场景 1-3 个例句。常用场景池：

- 代码评审 / 文档反馈 / 工作沟通
- 技术答疑 / 论坛提问
- 日常对话 / 客服 / 邮件

每条例句格式：

```
N. `English example sentence.`
   中文翻译（口语化，不要直译）。
```

### 7. 反向用法 / 变体（按需）

如果范式可以反向用（如「指出别人不足」也可以「主动声明自己范围」），加 `### 6. 反向用法` 一节，给 1 个例句。

否则省略本节。

### 8. 计算新章节编号

读取 `E:/work/blog/英语/tips.md`，扫描所有 `^## (.+?)、` 的标题，找出**最后一个**章节编号：

- 1-10 章用中文数字：`一、二、三、四、五、六、七、八、九、十`
- 11 章及以后用阿拉伯数字：`11、12、13、…`
- 当前最后一章是「十」→ 新章节为「11」
- 当前最后一章是「N」（N≥11）→ 新章节为「N+1」

新章节标题格式：

```
## <编号>、`<范式模板>` —— <一句话概括>
```

### 9. 写入文件

用 Edit 工具，把新章节插入到 `## 参考来源` 之前。

新章节固定结构（按需省略可选小节）：

```markdown
## <编号>、`<范式模板>` —— <一句话概括>

### 1. 例句

> <英文句子（修正后的）>
>
> <中文翻译>（潜台词：xxx）

### 2. 句子拆解

| 槽位 | 作用 | 本句对应 |
| --- | --- | --- |
| ...（每行一个槽位） |

核心句型：

\`\`\`
<范式模板>
\`\`\`

含义：「<一句话解释>」——<使用场景>。

### 3. 美国人口语同义表达

| 美国口语表达 | 语境 | 与原句的差异 |
| --- | --- | --- |
| \`...\` | ... | ... |

### 4. 语气梯度（可选）

| 词 | 语气 | 适用场景 |
| --- | --- | --- |
| ... |

### 5. 套用例子

#### <场景一>

1. \`English sentence.\`
   中文翻译。

#### <场景二>

...

### 6. 反向用法（可选）

> \`English sentence.\`
>
> 中文翻译。

---
```

注意：

- 新章节末尾留一条 `---` 分隔线
- 新章节插入位置：`## 参考来源` 这一行的**正前方**
- 即使原句被修正，文件中只写**修正后的版本**，不保留原错误句

### 10. 追加参考来源

用 WebFetch 或 WebSearch 找 1-2 条与本范式相关、可访问的英语学习资源链接，候选优先级：

1. Cambridge Dictionary / Merriam-Webster / Collins
2. English Stack Exchange / WordReference Forums
3. Grammarly Blog / British Council LearnEnglish

要求：

- **必须实际 WebFetch 验证可访问**，404/超时的链接禁止写入
- 若实在找不到合适来源，只追加 1 条；都找不到则跳过本步并在终端说明
- 追加位置：`## 参考来源` 列表末尾，格式 `- [标题](URL)`

### 11. md-zh 排版自检

调用 `md-zh` 技能获取规范，对**新写入的章节**逐项检查并修正：

- 中英文之间加空格（含与反引号、链接的边界）
- 中文与数字之间加空格
- 使用全角中文标点（，。：；！？）
- 引号用直角引号（「」『』）
- 专有名词大小写（GitHub、macOS、JavaScript 等）

如修正了内容，用 Edit 工具二次更新文件。

### 12. 终端汇报

输出简短报告：

```

   - 章节标题：## <编号>、`<模板>` —— <概括>
   - 美国口语同义：<N> 条
   - 套用场景：<N> 个
   - 参考来源：新增 <N> 条链接（已验证可访问）
   - md-zh 自检：通过 / 修正了 N 处
```

## 输出格式范例

参见 `E:/work/blog/英语/tips.md` 中第七章 `That [名词] only [动词] the [核心信息] for [对象/范围].`，所有新章节必须严格对齐其结构、标题层级与表格列名（**新章节多一节「美国人口语同义表达」，紧跟在「句子拆解」之后**）。

## 边界情况

| 情况 | 处理方式 |
| --- | --- |
| 原句完全没问题 | 跳过「⚠ 语法/用法纠正」段落，直接进入步骤 2 |
| 原句有多种合理修正 | 选最贴合潜台词的一种作为正式修正，其它候选在终端「说明」一行简要列出 |
| 美国口语同义表达找不到 2 条 | 至少给 1 条，并在终端说明「该范式过于书面/技术化，口语版本有限」 |
| 句子太短（< 3 词，如 `Got it.`） | 仍可分析，但范式只有 1-2 个槽位时省略「拆解表」，直接给套用例子 |
| 句子是固定习语（如 `Break a leg.`） | 提示老板「这是固定表达，没有可替换槽位」，征询是否仍要写入 |
| 句子已在 `tips.md` 中出现过 | 读文件全文检查 → 提示「该范式已存在第 X 章」，征询是否覆盖/补充 |
| 老板明确说「不要写入文件」 | 跳过步骤 9-12，只在终端输出完整章节内容 |
| `tips.md` 不存在 | 提示老板「目标文件不存在」，列出可能的备选路径，等待确认后再继续 |
| WebFetch 全部失败 | 跳过参考来源追加，在终端说明「本次未追加来源」 |

## 不做的事

- 不做整段翻译
- 不做发音/音标分析
- 不修改 `tips.md` 中已有的章节
- 不删除 `tips.md` 中任何内容
