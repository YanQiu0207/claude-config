---
name: code-review-local
description: |
    对本地代码目标（文件 / 目录 / 整个项目）进行多 agent 并行审核。
    5 个独立 reviewer 从 CLAUDE.md 合规、bug、安全、质量、测试 五个维度并行审查，按置信度阈值过滤后汇总成报告。
    同时输出到终端和 `.review-reports/*.md` 文件。
    当用户要求"审核本地代码"、"code review"、"多维度审查项目"、"评审这个目录"时触发。
argument-hint: "<path> [--focus=...] [--threshold=N] [--output=stdout|file|both]"
allowed-tools:
    - Read
    - Write
    - Bash
    - Glob
model: sonnet
---

当此 skill 生效时，回答第一行固定写：Using skill: code-review-local

## 输入

调用参数（位置 + 可选 flag）：

| 参数 | 说明 | 默认 |
|---|---|---|
| `<path>` | **必填**，审查目标：文件 / 目录 / 整个项目路径 | — |
| `--focus=a,b,c` | 只跑指定 reviewer，值来自 `compliance,bug,security,quality,test` | 全部 5 个 |
| `--threshold=N` | 置信度阈值（0-100） | `80` |
| `--output=X` | 输出去向：`stdout` / `file` / `both` | `both` |

调用示例：

- `/code-review-local E:\github\mykms`
- `/code-review-local E:\github\mykms --focus=bug,security --threshold=90`
- `/code-review-local src/auth.py --output=stdout`

如果用户用自然语言触发而未给 path，**必须先反问确认审查范围**，不要擅自扫描整个工作目录。

## 处理流程

### 1. 解析参数

从调用方 prompt 中提取 `<path>`、`--focus`、`--threshold`、`--output`。

- `<path>` 需转成 **绝对路径**（传给 subagent 时必须绝对路径，因为 subagent 是独立上下文）
- `--focus` 缺省时，5 个 reviewer 全跑
- `--threshold` 缺省 `80`
- `--output` 缺省 `both`

### 2. 前置检查

用 Bash 验证：

```bash
[ -e "<ABSOLUTE_PATH>" ] && echo OK || echo MISSING
```

若 MISSING，报错并退出：`❌ Target not found: <path>`。

### 3. 准备报告输出路径

- 若 `<path>` 是目录：`REPORT_DIR="<path>/.review-reports"`
- 若 `<path>` 是文件：`REPORT_DIR="<path 所在目录>/.review-reports"`

用 Bash 创建目录并写一个自忽略的 .gitignore（幂等，不覆盖已存在的）：

```bash
mkdir -p "<REPORT_DIR>"
[ ! -f "<REPORT_DIR>/.gitignore" ] && printf '*\n' > "<REPORT_DIR>/.gitignore"
TS=$(date +%Y%m%d_%H%M%S)
echo "<REPORT_DIR>/code-review_${TS}.md"
```

记住这个 `REPORT_PATH` 和 `TS`，后面写文件和报告头要用。

### 4. 并行派发 reviewer（后台执行）

**CRITICAL**：所有 Agent 工具调用 **必须放在同一条消息（同一个 tool-call batch）里同时发起**，否则它们会串行执行，丧失并行收益。

对每个要跑的 reviewer，在同一条消息里发一个 Agent 调用：

```javascript
Agent({
    description: "review: <dimension>",
    subagent_type: "reviewer-<dimension>",   // compliance / bug / security / quality / test
    run_in_background: true,
    prompt: "请对以下目标执行代码审查。\n\n" +
            "TARGET_PATH: <ABSOLUTE_PATH>\n" +
            "CONFIDENCE_THRESHOLD: <THRESHOLD>\n\n" +
            "按你的 agent 定义中的流程和输出格式返回 Markdown 报告，不要输出其他说明文字。"
})
```

- 使用 `run_in_background: true`：所有 reviewer 同时在后台并行执行，完成后通过 task-notification 将结果发回主会话
- 5 个 reviewer 对应 5 个 Agent 调用；`--focus` 指定时只发对应的子集
- 同一条消息里并列发起
- 派发后立即告知用户"已启动 N 个 reviewer，后台执行中…"，无需阻塞等待

### 5. 汇总并渲染报告

所有 reviewer 返回后，拼装成一份报告：

````markdown
# Code Review Report

- **Target**: `<path>`
- **Time**: YYYY-MM-DD HH:MM:SS
- **Reviewers ran**: compliance, bug, security, quality, test
- **Confidence threshold**: 80
- **Generated by**: code-review-local skill

## Summary

扫完后用 1-3 句总结整体情况（哪些维度问题最多、最关键的 1-2 个风险）。

---

（按固定顺序拼接各 reviewer 返回的 Markdown 章节：compliance → bug → security → quality → test）

<reviewer-compliance 的原始输出>

---

<reviewer-bug 的原始输出>

---

...
````

注意：

- 各 reviewer 的输出已经自带章节标题（`## 📘 CLAUDE.md Compliance` / `## 🐛 Bugs` 等），直接拼接即可，不要再加一层
- 用 `---` 分隔不同 reviewer 的章节
- 如果某个 reviewer 失败（返回报错），在对应位置用 `## ⚠ <dim> — 运行失败` 占位，并附错误信息；不要因为一个失败就中断整个流程

### 6. 输出

根据 `--output`：

- `stdout` / `both`：直接把完整 Markdown 报告文本打印到对话里
- `file` / `both`：用 Write 工具把同样的内容写入 `REPORT_PATH`

写完文件后，在对话末尾追加一行：

```
📄 Report saved: <REPORT_PATH>
```

## 输出格式（终端最终响应）

首行：`Using skill: code-review-local`

之后分两个阶段：

**派发阶段**（步骤 4 完成后立即输出）：
- 一行简短状态，如 `已在后台启动 5 个 reviewer，执行中…`

**汇总阶段**（所有 task-notification 到齐后输出）：
1. 完整报告内容（仅 `--output != file` 时）
2. `📄 Report saved: <path>`（仅 `--output != stdout` 时）

## 边界情况

- **path 不存在**：报错退出
- **path 是空目录**：跑 reviewer 时也会报"无源文件"，如实汇总，报告仍生成
- **某个 reviewer 超时或异常**：占位为"⚠ 运行失败"，其他维度照常返回
- **`--focus` 含未知值**：只忽略未知值，对已知值继续跑；若全部未知，报错退出
- **`--threshold` 非 0-100 整数**：报错退出
- **审查目标包含海量文件**：reviewer agent 自己有抽查策略，skill 不做文件预过滤
- **路径含空格**：传给 subagent 的 TARGET_PATH 不用引号包裹，在 Bash 中涉及到该变量的地方用 `"..."` 引用

## 硬约束

- **不要在 skill 层再次过滤 confidence**：reviewer 已按阈值过滤，skill 只做拼接
- **不要修改 reviewer 的输出内容**：保持原文，最多在最外层加 Summary
- **不要派生非 reviewer-* 的 agent**：本 skill 只调用 5 个官方 reviewer
- **subagent 是独立上下文**：Agent prompt 中必须给绝对路径和阈值，不能假定 subagent 能看到本对话
