# 开发导航地图（dev-map）

> 偏代码落点的导航：回答「改这个功能该动哪些文件、会牵动什么」。与 `overview.md`（架构全貌）互补。
> 维护：谁动代码谁更新——改码前先查、落点变化后回写。仓库大时按子系统拆分。

## 功能 → 落点

| 功能 / 关注点 | 主要落点文件 | 说明 |
|--------------|------------|------|
| 示例：用户登录 | `src/auth/login.py`、`src/auth/session.py` | 入口在 login，会话状态在 session |

## 配置定义位置

| 配置类别 | 定义位置 |
|---------|---------|
| 示例：超时 / 重试 | `src/config/defaults.py` |

## 影响链路（改 X 会牵动什么）

| 改动点 | 下游影响 |
|-------|---------|
| 示例：改 `Session` 结构 | `login.py`、`middleware/auth.py`、所有读 session 的 handler |

## 既有写法 / 惯例

- 示例：错误统一用 `Result[T]` 返回，不抛异常
- 示例：新增配置项必须在 `defaults.py` 给默认值，并在 `config.md` 注明
