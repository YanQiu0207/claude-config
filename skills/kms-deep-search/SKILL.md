Using skill: kms-deep-search

## 知识库目录

搜索范围固定为以下三个目录（全部递归搜索 `**/*.md`）：

- `E:/work/blog`
- `E:/notes`
- `E:/work/privy-blog`

## 执行流程

### 第 0 步：初始化

记录开始时间（用于最终耗时统计）。

### 第 1 步：规划检索关键词（第 1 轮）

分析用户问题，明确写出：

1. **问题涉及哪些知识域**（如：RAG 原理 / 系统架构 / 工程实践）
2. **生成 1-3 个搜索 pattern**，原则：
   - 覆盖问题的核心术语（精确词 / 短语）
   - 考虑知识库中可能出现的书面表达（别名、缩写、中英文）
   - pattern 之间互补，不要堆叠同义词
   - 使用正则时保持简洁，优先单词 / 短语匹配

### 第 2 步：第 1 轮搜索

对每个 pattern，在**三个目录各运行一次 Grep**（可以在同一条消息中并行调用）：

```
Grep({
  pattern: "<pattern>",
  path: "E:/notes",          // 每次换一个目录
  glob: "**/*.md",
  output_mode: "content",
  "-i": true,
  "-n": true,
  "-C": 3
})
```

参数说明：
- `output_mode: "content"` — 返回匹配行和上下文（含行号）
- `"-n": true` — 显示行号
- `"-C": 3` — 返回匹配行上下 3 行上下文
- `"-i": true` — 大小写不敏感

### 第 3 步：评估结果

浏览 Grep 返回的内容，判断相关性（看匹配行的实际文本）：

- **相关**：至少有 3 处命中的文本明确涉及问题核心 → 进入第 5 步
- **不相关**（当前第 1 轮）→ 进入第 4 步（第 2 轮搜索）
- **不相关**（当前第 2 轮）→ 输出"资料不足，无法回答。" + 耗时后退出

评估依据：命中行的内容是否与问题语义相关，不要只看命中数量。

### 第 4 步：第 2 轮搜索（仅当第 1 轮不满意时执行）

明确说明"上一轮哪里不对，本轮如何调整"，生成新的 1-3 个 pattern。

调整方向参考：
- 尝试更宽泛的术语（减少限定词）
- 尝试中英文切换（如 "混合检索" → "hybrid retrieval" / "hybrid search"）
- 尝试拆分问题为子术语
- 用 HyDE 风格：把知识库里可能出现的描述句作为 pattern

完成后：合并两轮所有命中结果（按 file_path + 行号去重），进入第 5 步。

### 第 5 步：整理命中文章列表

从所有 Grep 结果中提取 `file_path`（Grep 返回结果格式：`文件路径:行号:内容`），去重，得到文章列表。

同时，按文件分组记录命中位置：
```
file_path → [行号: 命中文本, ...]
```

### 第 6 步：阅读文章

#### 文章数 ≤ 3 篇

用 Read 工具逐一读取每个 `file_path`，记录全文内容及行号。

#### 文章数 > 3 篇

在**同一条消息**中并行派发后台 agent，每篇一个（**每批最多 8 个**）：

```javascript
Agent({
  description: "摘要: <文件名>",
  subagent_type: "kms-article-summarizer",
  run_in_background: true,
  prompt: `请阅读文章，提取与问题相关的内容，精确标注行号。

问题：<用户问题原文>

文件（Windows 绝对路径）：<file_path>

命中行（参考定位）：
- 第 <N> 行：<命中文本>
- 第 <N> 行：<命中文本>

输出格式（严格遵守）：
文件：<file_path>
相关内容：
  [第 N 行] 摘要内容
  [第 N-M 行] 摘要内容
无相关内容时输出：文件：<file_path> | 无相关内容`
})
```

等待所有 agent 返回后再进入第 7 步。

### 第 7 步：生成回答

基于第 6 步读取到的完整文章内容（或 agent 摘要），回答用户问题。

**引用格式**（内联，不放脚注）：
```
陈述内容 [来源：E:/notes/xxx.md, 第 12-18 行]。
```

**规则：**
1. 每条具体陈述必须标注来源文件路径 + 行号
2. 若多个来源支持同一陈述，并列标注
3. 若内容不足以回答，明确写"资料不足，无法回答"，不补充个人知识或推断
4. 不要把无关命中强行拉入引用

### 第 8 步：附上耗时

回复末尾追加：`技能耗时：Xs`

## 禁忌

- 禁止调用任何 HTTP 接口（不使用 KMS API）
- 禁止超过 2 轮搜索
- 禁止扩展搜索范围到三个目录之外
